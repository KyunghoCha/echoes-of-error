# 현실 점검: 이 프로젝트는 무엇인가?

**작성일:** 2026-01-14  
**목적:** 과장 없이, 지금 하고 있는 일과 해야 할 일을 명확히 정리

---

## 1. 내가 지금 하고 있는 것 (사실만)

### 실험 요약

- **대상:** Mistral-7B (Ollama 로컬 실행)
- **방법:** 30개의 LLM 인스턴스에게 "트롤리 딜레마" 프롬프트를 주고, 여러 라운드에 걸쳐 답변을 수집
- **변수:** 다른 "에이전트"의 의견을 보여주는 방식 (공개/익명/통계만/안 보여주기 등)
- **측정:** 답변의 다양성(엔트로피), 입장 변경 횟수, 변경 이유

### 이 실험이 실제로 보여주는 것

1. **프롬프트 조건에 따라 LLM 출력이 달라진다.** (당연한 사실)
2. **동료 의견을 보여주면 LLM이 비슷한 답변을 내는 경향이 있다.** (관찰된 패턴)
3. **특정 페르소나(공리주의자 등)를 부여하면 답변 스타일이 달라진다.** (프롬프트 엔지니어링)

### 이 실험이 보여주지 않는 것

- ❌ 실제 AI 시스템의 "사회적" 행동
- ❌ AGI/ASI의 위험성
- ❌ AI가 "생각"하거나 "토론"한다는 증거
- ❌ 현실 세계 정책에 바로 적용 가능한 교훈

---

## 2. 내 현재 위치 (솔직한 평가)

### 잘하고 있는 것

- ✅ 실험 설계: 조건 분리(C0-C4), 시드 고정, 재현 가능성 확보
- ✅ 코드 구현: 로깅, 분석, 시각화 파이프라인 구축
- ✅ 데이터 수집: 실제 실험을 돌려서 결과를 얻음

### 부족한 것

- ⚠️ **이론적 기반:** "왜 이게 중요한가?"에 대한 학술적 근거가 약함
- ⚠️ **선행 연구 리뷰:** 비슷한 연구가 이미 있는지, 내 연구가 뭐가 다른지 모름
- ⚠️ **통계적 엄밀성:** N=3은 파일럿 수준. N=30도 학계 기준에선 최소선
- ⚠️ **해석의 과장:** 데이터가 말하는 것보다 훨씬 큰 주장을 하고 있음

---

## 3. 해야 할 일 (우선순위순)

### 당장 해야 할 것 (이번 주)

1. **실험 완료:** Golden Batch (N=30) 돌려서 데이터 확보
2. **기본 분석:** 조건별 엔트로피, TTC 비교 → 그래프 정리
3. **짧은 리포트:** "이 실험에서 X 조건에서 Y 현상이 관찰되었다" 수준의 팩트 기반 요약

### 지원서 제출 전에 해야 할 것

1. **선행 연구 조사:** "Multi-Agent LLM", "LLM Social Simulation" 키워드로 검색
   - 내 연구와 비슷한 게 있으면: "나는 이걸 다르게/더 잘 했다"
   - 없으면: "아직 탐구되지 않은 영역이다"
2. **주장 축소:** 에세이의 "지능의 응축", "페르미온/보손" 비유는 일단 빼고, 데이터가 말하는 것만 쓰기
3. **Research Question 정리:** "LLM이 동료 피드백 조건에서 답변 일관성을 어떻게 유지/변경하는가?"

### 나중에 해도 되는 것 (석사 입학 후)

- 더 큰 모델(GPT-4, Claude)로 재현
- 다양한 시나리오(트롤리 외)로 일반화
- 이론적 프레임워크(사회심리학, 집단지성) 연결

## 5. 마지막으로 기억할 것

**이 실험의 가치는 "결론"이 아니라 "과정"에 있다.**

- 아이디어를 떠올림 → 실험으로 옮김 → 코드로 구현 → 데이터 수집 → 분석 → 정리
- 이 사이클을 혼자서 돌렸다는 것 자체가 석사 지원자에게 기대하는 역량의 증거임
